### ❓ 当前疑问
- Go的interface相较于Java的有何区别
  - Go 接口通过方法匹配而非类型声明实现了解耦，使得实现方无需声明实现、无需改接口、无需侵入结构。
    而 Java 的继承关系要求显式绑定、全局声明，一旦接口变化，所有实现方都受到牵连。
- 接口组合的应用是
  - 将大接口拆成多个小接口，按需组合，实现更灵活、解耦、可扩展的程序结构。
- 断言是否会影响性能
  - 不会,类型断言则是在保持高性能的前提下动态识别这些能力
